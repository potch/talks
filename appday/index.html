What's a Firefox OS App, Anyway??

The Manifest.

  • A JSON file contaning metadata about your app
  • name, icon, etc.
  • application/x-web-app-manifest+json
{
    "name": "TicTacApp",
    "description": "A simple tic tac toe game",
    "developer": {
        "name": "Potch",
        "url": "http://github.com/potch"
    },
    "launch_path": "/index.html",
    "default_locale": "en",
    "icons": {
        "128": "/icon-128.png"
    }
}

Kinds Of Firefox OS Apps

"Hosted App"

Existing Web Site + App Manifest = Hosted App

Pros

  • Easy. takes very little work to convert!
  • Works well with existing back-end frameworks (Django, Rails, etc.)
  • Great for developers who aren't comfortable with front-end tech.

Cons

  • Not that easy.
    • Where's the back button?
  • Performance
    • Synchronous page-loads on every user action = slow.
    • Bandwidth hog!
      • Requires WiFi to work well
      • Bandwidth === $$$
  • Little-to-no offline support.
    • Browser cache might save you...
  • Poor first-run experience
    • Gray screen of sadness
  • "Glorified Bookmark"
    • :(

"The SUV"

"Packaged App"

.zip of Static resources (including App Manifest)

Pros

  • No Hosting Required!
    • HTML/CSS/JS nerds, rejoice!
  • Awesome first-run experience!
    • Full .zip is downloaded when the user installs the app
  • Performance
    • No network requests for resources
  • No Connection? No Problem.
  • API Goodies
    • Socket connections, SMS, Camera, Contacts
  • "Feels Like Native!"
    • <3

Cons

  • Monolithic updates
  • Communication with a server is harder
    • CORS
  • Much stricter review process
    • You have access to more powerful APIs — we need to know that you're playing nice.
  • Well, it's not really a Web App, is it?
    • Packaged apps only run where they are supported
      • Firefox OS, Firefox For Android

"The Bullet Train"

"That actually sounds pretty great, but I do have a server-side component"

What if I told you there was a third way?

"Modern HTML5 Web App"

Pros

  • Awesome first-run experience!
  • It's still a web app
    • Dynamic content
  • Performance
    • No network requests for static resources
  • Graceful Offline Experience
  • Bandwidth-Friendly
  • It's still a web site!
    • Available wherever fine websites are served
  • "Feels like Native!*†‡"

Cons

  • Not your granddaddy's web app.
    • Requires knowledge of new and emerging Web tech
  • Fewer API goodies
    • Privileged APIs are not available
    • More are coming!
  • Heavy on the JavaScript

"Well okay, that last one sounds complicated"

But totally worth it.

Secret Ingredients

  • Designed for Mobile
  • Touch-friendly
  • Static resources stored locally
  • Handles connectivity loss gracefully
  • Loads content/data asynchronously
  • Supports deep linking!
